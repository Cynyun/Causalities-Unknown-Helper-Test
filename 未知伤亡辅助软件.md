# Causalities-Unknown-Helper-Test

## 1. 项目概述
**Causalities-Unknown-Helper-Test** 是一款专为特定游戏设计的**离线优先、极致轻量**的桌面辅助工具。项目采用 **Tauri 2.x + Vue 3 + Vite** 构建，旨在为玩家提供毫秒级响应的本地知识库查询与数据计算服务。

当前版本（v0.1）以 **本地 Wiki 内容展示** 为核心功能，通过内置 Python 爬虫自动同步游戏数据；未来将扩展至 **合成配方树可视化、资源规划计算器** 等深度辅助功能。整体设计遵循“工具属性优先”原则，拒绝臃肿，安装包体积控制在 10MB 以内，内存占用低于 60MB。

## 2. 核心技术栈

| 层级 | 技术选型 | 选型理由 |
| :--- | :--- | :--- |
| **桌面框架** | Tauri 2.x | 系统原生 WebView，打包体积极小，Rust 后端保障数据安全与高性能 I/O |
| **前端框架** | Vue 3 + Vite | 组合式 API 适合复杂交互逻辑，Vite 构建速度快，HMR 体验佳 |
| **样式方案** | UnoCSS + Naive UI | 按需生成零冗余 CSS，Naive UI Tree-shaking 友好，适配暗色模式 |
| **数据渲染** | markdown-it + Shiki | 纯文本着色无 DOM 开销，支持 Wikitext/Markdown 混合解析 |
| **数据获取** | Python Sidecar | 复用现有爬虫生态，通过 Tauri Sidecar 机制无缝集成 |
| **本地存储** | SQLite (FTS5) | 嵌入式全文检索，支持合成表多对多关系查询，零网络依赖 |
| **状态管理** | Composables + Pinia | 轻量数据用组合式函数直读文件，全局配置/缓存用 Pinia |

## 3. 功能模块规划

### ✅ Phase 1：本地 Wiki 引擎（当前重点）
-   **数据采集**：内置 Python 爬虫 Sidecar，支持一键增量更新 Wiki 数据至本地 SQLite
-   **内容渲染**：安全解析 Wikitext/HTML，内部链接路由拦截，图片懒加载 + 离线占位符
-   **全文检索**：基于 SQLite FTS5 的毫秒级搜索，支持拼音/别名匹配
-   **阅读体验**：虚拟滚动长列表、暗色模式跟随系统、历史记录与书签收藏

### 🔜 Phase 2：合成表 / 科技树辅助（后期扩展）
-   **配方可视化**：交互式 DAG 图展示物品合成路径，支持缩放/拖拽/节点高亮
-   **需求反推**：输入目标物品数量，自动递归计算所需原材料及中间产物
-   **产线规划**：结合游戏内采集速率，估算生产时间瓶颈
-   **数据联动**：合成表数据与 Wiki 条目双向跳转，点击材料直接查看详细信息

### 💡 Phase 3：进阶辅助（远期愿景）
-   游戏存档解析与进度追踪
-   自定义标注与笔记系统
-   Mod 兼容性检测提示

## 4. 架构设计亮点

### 🗂️ 数据与 UI 彻底解耦
```text
[Vue 前端] ←IPC→ [Tauri Rust 后端] ←文件/DB→ [SQLite / JSON]
      ↑                                      ↑
  纯展示层                                数据层
  (无业务逻辑)                    (Python 爬虫独立写入)
```
-   前端**不发起任何 HTTP 请求**，所有数据通过 Tauri IPC 同步读取本地文件/数据库
-   Python 爬虫作为独立 Sidecar 进程运行，崩溃不影响主界面
-   数据更新无需重新打包应用，替换数据文件即可生效

### ⚡ 轻量化保障措施
-   **禁用 Nuxt/SSR**：纯 SPA + Hash 路由，避免 Node.js 运行时开销
-   **虚拟滚动强制启用**：所有列表页超过 50 条即触发 `vue-virtual-scroller`
-   **代码分割策略**：Wiki 渲染器、合成图引擎、搜索模块按需异步加载
-   **图片容错管线**：未下载图片自动降级为带 Caption 的 SVG 占位符，杜绝破碎图标

## 5. 开发规范与约束

-   **离线优先**：所有核心功能必须在完全断网状态下可用
-   **性能预算**：首屏渲染 < 300ms，搜索响应 < 50ms，滚动帧率 ≥ 55fps
-   **体积红线**：最终 NSIS 安装包 ≤ 10MB，便携版 exe ≤ 8MB
-   **数据格式契约**：Python 爬虫输出必须遵循预定义的 JSON Schema / SQLite DDL，前后端以此为唯一接口
-   **无障碍基础**：支持键盘导航、语义化标签、高对比度模式

## 7. 为什么不是 Electron / Web App？

| 考量维度 | GameHelper (Tauri) | Electron | PWA / Web App |
| :--- | :--- | :--- | :--- |
| 安装包体积 | **~6 MB** | ~180 MB | 无需安装 |
| 内存占用 | **~45 MB** | ~200 MB | 受浏览器限制 |
| 本地文件访问 | **原生高性能** | Node.js fs | 受限 / 需用户授权 |
| 离线可靠性 | **✅ 完全离线** | ✅ 完全离线 | ❌ 缓存不可靠 |
| 系统集成 | **托盘/通知/快捷键** | 同左 | ❌ 几乎无 |
| 合成图渲染性能 | **Canvas/WebGL 原生** | 同左 | 受沙箱限制 |

> 📌 **项目定位重申**：这不是一个“套壳网站”，而是一个**真正的本地原生工具**。所有技术决策均服务于“快、小、稳”三个字。Wiki 是起点，而非终点——架构已为合成树、数据计算等重交互模块做好铺垫。
